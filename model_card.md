# Phi-3-mini Medical Triage Assistant

Fine-tuned from `microsoft/Phi-3-mini-4k-instruct` using QLoRA on a
medical triage instruction dataset to classify patient symptom
presentations into four urgency levels.

## Model Details

| Property | Value |
|---|---|
| Base model | microsoft/Phi-3-mini-4k-instruct (3.8B) |
| Fine-tuning method | QLoRA — 4-bit NF4 quantisation + LoRA r=16 |
| Training platform | Kaggle T4 GPU (16GB VRAM) |
| Training examples | 200 synthetic patient presentations |
| LoRA target modules | qkv_proj, o_proj |
| Training epochs | 5 |

## Intended Use

Educational and portfolio demonstration of domain-specific LLM
fine-tuning. **NOT for clinical deployment or real medical decisions.**
Always consult a qualified healthcare professional.

## Training Data

200 synthetic instruction-tuning examples generated using Claude
(Anthropic), covering four triage levels with realistic patient
presentations including varied ages, genders, and symptom descriptions.
Each example follows the Alpaca instruction format.

| Triage Level | Training Examples |
|---|---|
| EMERGENCY | 50 |
| URGENT | 50 |
| SEMI-URGENT | 50 |
| NON-URGENT | 50 |

## Evaluation Results

Evaluated on 100 held-out synthetic patient presentations (25 per class).

| Model | Triage Accuracy | ROUGE-L | EMERGENCY Recall |
|---|---|---|---|
| **This model (fine-tuned)** | **64%** | **0.42** | **100%** |
| Base Phi-3-mini (zero-shot) | 37% | 0.21 | 100% |

**Key finding:** The fine-tuned model achieves 64% overall accuracy,
outperforming the base model by 27 percentage points. EMERGENCY recall
is 100% — the model never misses a genuine life-threatening emergency.
The main remaining weakness is the URGENT class, which is frequently
predicted as EMERGENCY — a conservative error that is clinically safer
than the reverse.

## How to Use
```python
from transformers import AutoModelForCausalLM, AutoTokenizer, pipeline
import torch

model_id = "Milon96/phi3-medical-triage"

tokenizer = AutoTokenizer.from_pretrained(model_id, trust_remote_code=True)
model = AutoModelForCausalLM.from_pretrained(
    model_id,
    device_map="auto",
    torch_dtype=torch.float16,
    trust_remote_code=True,
)

pipe = pipeline("text-generation", model=model, tokenizer=tokenizer)

instruction = """You are a medical triage assistant. Classify the following
patient presentation into: EMERGENCY, URGENT, SEMI-URGENT, NON-URGENT."""

input_text = "72-year-old male with sudden chest pain radiating to his left arm, sweating, nausea. Onset 15 minutes ago."

prompt = f"<|system|>\n{instruction}<|end|>\n<|user|>\n{input_text}<|end|>\n<|assistant|>\n"

result = pipe(prompt, max_new_tokens=150, do_sample=False, temperature=1.0)
response = result[0]["generated_text"].split("<|assistant|>")[-1].replace("<|end|>","").strip()
print(response)
```

## Limitations

- Trained on synthetic data generated by Claude — not validated on
  real clinical cases
- URGENT class has low accuracy (4%) — frequently over-triaged to EMERGENCY
- Not suitable for real medical decision-making
- English language only
